## Description
A hierarchical table is used in order to showcase structured data on multiple levels (p.ex: canton > district > municipality). It is helpful to analyse the repartition or organisation of a phenomena in relation to its structure.

Lines are used to represent observations, organized in a hierarchical manner. Columns are assigned a variable each.

## Do's and don'ts
- Not too many columns (* MH : how many maximum*)
- Not too many hierarchical levels (* MH : how many maximum*)
- One subject per table
- No cell fusion to guarantee machine readability
