## Points of attention
- Not too many columns (3 to 5 columns maximum, this is in order to pay attention to mobile compatibility)
- No cell fusion to guarantee machine readability
- The number of rows in a table should not exceed 20. There are some exceptions to this rule such as for a table where each row is a swiss canton.
- The variations documented for the tables guidelines can be subject to some changes depending on the objective of the table. The positionning of the variations element such as the total and the highlight can be changed depending on the usage.
- Only one subject should be represented per table.
