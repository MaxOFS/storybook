# Introduction
With the new concept, all FSO graphics will be standardized. In principle, there is no longer any difference between print and web graphics. Of course, there are exceptions, such as the Treemap Chart or Marimekko Chart, which are difficult to reproduce ideally in print.For all graphics, a reference to the source and the copyright are mandatory.The height of the visualization remains variable, the width is to be adapted to the respective platform on which the graphics are published. See [Accessibility](?path=/docs/cross-media-introduction--docs).

## Provide figures as accurately as possible
Even if a graphic often shows strongly rounded values, every graphic should always be calculated with the maximum available accuracy. This requires that the data supplied by the sections have at least three numbers after the decimal point. Depending on the graph model, the values are output more or less rounded. Numbers already rounded by the section are thus rounded again, which could lead to differences in the results.

## Colors
Only the color palettes developed specifically for the graphics are to be used. The new color scheme is intended to create a clear recognizability of the FSO graphics, which is continuous throughout all platforms. See [Colors](?path=/docs/corporate-colors-introduction--docs).
## Graphic Number
All assets are registered with a standardised FSO number. This is available in the metadata or at the bottom of the diagram (web).

In publications, all graphic models (diagrams, maps, schemes) are numbered together with G-numbers (G 1.1, G 1.2, G 1.3, G 1.3A, G 1.3B…). Whether these are numbered chapter by chapter or holistic numbering is done is decided by the author or the publication model.
Numbering is mandatory in standard publications and in the statistical yearbook, optional in all other FSO publications.
