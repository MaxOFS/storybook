# Accessibility for Colors
Color should not be used as the only visual variable to communicate an information or to distinguish a visual element.
If colour is used to communicate information, it must be accessible to all users, including users with colour difficulties or limited vision. 

It is therefore essential to have a sufficient contrast and colour palettes that are also accessible to people with colour blindness. 
## Contrast
According to the WCAG 2.1 guidelines, the contrast must be at least 3:1. 
## Color tools
In order to generate an accessible colour scale, the following tools can be used:
### Adobe colour
As a first step, it is important to pick colours that provide a minimun contrast of 3:1. The Adobe Color Accessibility Tool displays the contrast and provides a preview of the colour in action for graphic components.

Link to Adobe Color: [https://colour.adobe.com/fr/create/colour-contrast-analyzer](https://colour.adobe.com/fr/create/colour-contrast-analyzer)
### VizPalette
As soon as colours with enough contrast have been picked, the VizPalette tool is a great tool, to see how the different colours interact with each other in combination and how they look in action by generating the different views for:

Link to VizPalette: [https://projects.susielu.com/viz-palette](https://projects.susielu.com/viz-palette)

### Chroma.js
In order to create sequential or diverging colours Chroma.js is a great tool to mastering multi-hued, multi-stops colour scales.

Link to Chroma.js: [https://gka.github.io/chroma.js/](https://gka.github.io/chroma.js/)
### Adobe Illustrator
Adobe Illustrator tool allows to visualize the window with Protanopia or Deuteranopia colourblindness (View > Proof Setup)
